## 三、关键发现

1. **流式缺少 `finish_reason:"stop"` 终止块（OpenAI 兼容缺口）**
   实测流式响应所有 chunk 均为 `finish_reason:null`，直接以 `data: [DONE]` 结束，缺少
   标准 OpenAI 的终止块 `{"choices":[{"delta":{},"finish_reason":"stop"}]}`。
   部分 OpenAI SDK 依赖该字段判断"正常结束 vs 被截断"。
   - 当前处理: 回归测试校验真实契约(增量内容 + `[DONE]` 终止)，并对 `finish_reason`
     做**前向兼容**断言(`in (None,"stop")`)，deno_pro 修复后自动转绿。
   - **建议**: 后续在 `deno_pro.ts` 流式分支于 `[DONE]` 前补发一个 `finish_reason:"stop"` 终止块。

2. **图像生成偶发瞬时失败**（背靠背连发尤甚）
   单独连发 3 次对话触发图像 3/3 成功(返回 `![图片](https://file-cdn.sider.ai/...)`)，
   但端点测试与对话触发测试相邻执行时出现过一次"图片生成刚刚失败了"(deno_pro 优雅兜底)。
   - 当前处理: 图像测试加 `retry`(默认3次/间隔4s)以稳定回归。
   - 说明: 这是上游 sider.ai 的偶发性，非 deno_pro 缺陷。

3. **个别模型响应极简**: `claude-opus-4.8` 对"介绍杭州"仅返回 4 字符。不影响测试(仅断言非空)，
   但提示性能字符数横比时需注意模型措辞差异。

4. **环境注意点**: Windows 控制台需 `PYTHONIOENCODING=utf-8`(主页含 emoji/中文)；
   pip 读取含中文注释的 requirements 会 GBK 报错，故依赖文件注释保持 ASCII；
   anaconda 的 urllib 因 certifi 过期会证书校验失败，但 **`requests`(verify=True) 正常**，框架用 requests。

## 四、使用方式

```bash
# 安装
d:/Anaconda3/envs/python310/python.exe -m pip install -r requirements-test.txt

# 在仓库根目录运行 (Windows 先 set PYTHONIOENCODING=utf-8)
pytest -m smoke                 # 零额度冒烟
pytest -m "not perf"            # 功能回归(代表子集)
pytest --full -m "not perf"     # 全量 44 模型
pytest -m perf                  # 性能
pytest --base-url http://localhost:8000   # 切到本地 deno 实例(本地 TDD 回归)
```
